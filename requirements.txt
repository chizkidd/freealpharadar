# FreeAlphaRadar — runtime dependencies (slim, Streamlit Cloud friendly).
# Zero API keys, zero secrets. Every data source is free and key-less.
#
# This is the ONLY file Streamlit Community Cloud installs automatically. It is
# intentionally lightweight so the free tier (~1 GB RAM) builds fast and never
# OOMs. The optional FinBERT / XGBoost stack lives in requirements-ml.txt; when
# it is absent the app degrades gracefully to a deterministic lexicon sentiment
# backend and rule-based scoring with no loss of functionality.
#
#   pip install -r requirements.txt        # run the app (lexicon sentiment)
#   pip install -r requirements-ml.txt     # optional: real FinBERT + XGBoost
#   pip install -r requirements-dev.txt    # optional: tests + formatting
#
# ---------------------------------------------------------------------------
# Version strategy: floors, not hard pins, on the *compiled* packages.
#
# Streamlit Community Cloud picks the Python version itself (currently 3.14),
# and a hard pin like `pandas==2.2.3` / `lxml==5.3.0` has NO prebuilt wheel for
# the newest Python -> pip falls back to building from source, which times out
# or fails (lxml needs libxml2 headers, pandas needs a C/Cython toolchain) and
# the deploy never starts. Using `>=` floors lets pip resolve whatever recent
# version ships a prebuilt wheel for the interpreter in play, so the app builds
# fast and starts with ZERO manual settings changes on any Python (3.11–3.14).
# Upper caps (<next-major) guard against surprise breaking releases. The code is
# verified green against both the floors and the latest majors (pandas 3.x /
# numpy 2.x) by the test suite.
